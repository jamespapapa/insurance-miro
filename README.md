## ⚡ 프로젝트 개요

**MiroFish** 는 다중 에이전트 기술 기반의 차세대 AI 예측 엔진입니다. 현실 세계의 씨앗 정보(예: 속보, 정책 초안, 금융 신호)를 추출해 고충실도의 평행 디지털 세계를 자동으로 구축합니다. 이 공간에서 독립적인 인격, 장기 기억, 행동 논리를 갖춘 수만 개의 에이전트가 자유롭게 상호작용하며 사회적으로 진화합니다. 당신은 ‘신의 시점’에서 변수를 동적으로 주입해 미래의 흐름을 정밀하게 시뮬레이션할 수 있습니다——**미래를 디지털 샌드박스에서 미리 리허설하고, 수많은 모의전 끝에 의사결정이 승리하도록 돕습니다**.

> 당신이 할 일: 씨앗 자료(데이터 분석 보고서 또는 흥미로운 소설 이야기)를 업로드하고, 자연어로 예측 요구를 설명하기</br>
> MiroFish가 제공: 상세한 예측 보고서, 그리고 깊이 있게 상호작용 가능한 고충실도 디지털 세계

### 우리의 비전

MiroFish는 현실을 매핑하는 군집 지능의 거울을 구축하는 데 전념하며, 개인 간 상호작용이 촉발하는 집단적 창발을 포착해 기존 예측의 한계를 돌파합니다:

- **거시적으로**: 우리는 의사결정자를 위한 리허설 실험실로, 정책과 PR이 무위험 환경에서 시행착오를 할 수 있게 합니다
- **미시적으로**: 우리는 개인 사용자를 위한 창의적 샌드박스로, 소설 결말을 추론하든 상상을 탐험하든 모두 재미있고, 잘 놀 수 있고, 손에 닿을 만큼 가깝습니다

진지한 예측부터 재미있는 시뮬레이션까지, 우리는 모든 ‘만약’의 결과를 보이게 하며, 만물을 예측하는 것을 가능하게 합니다.

## 🚀 빠른 시작

### 1. 소스 코드 배포(추천)

#### 사전 요구사항

| 도구 | 버전 요구사항 | 설명 | 설치 확인 |
|------|---------|------|---------|
| **Node.js** | 18+ | 프론트엔드 실행 환경, npm 포함 | `node -v` |
| **Python** | ≥3.11, ≤3.12 | 백엔드 실행 환경 | `python --version` |
| **uv** | 최신 버전 | Python 패키지 관리자 | `uv --version` |

#### 1. 환경 변수 설정

```bash
# 예시 설정 파일 복사
cp .env.example .env

# .env 파일을 편집해 필요한 API 키를 입력
```

**필수 환경 변수:**

```env
# LLM API 설정(OpenAI SDK 형식의 어떤 LLM API도 지원)
# 알리 바일리엔 플랫폼 qwen-plus 모델 사용 권장：https://bailian.console.aliyun.com/
# 소모가 큰 편이니, 우선 40회 미만의 시뮬레이션을 시도해 보세요
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Zep Cloud 설정
# 월간 무료 할당량만으로도 간단한 사용을 지원 가능：https://app.getzep.com/
ZEP_API_KEY=your_zep_api_key
```

#### 로컬 OpenCode Serve 사용(OpenAI OAuth)

이미 로컬 OpenCode에서 OAuth로 OpenAI를 연결해두었다면, OpenAI API Key를 MiroFish에 직접 노출하지 않아도 되며,
대신 MiroFish가 로컬 `opencode serve` + 저장소 내장 OpenAI-compatible 프록시를 통해 호출하도록 할 수 있습니다.

1. 이 저장소 루트 디렉터리의 `.env`에서 `LLM_*`를 아래 세트로 변경합니다:

```env
LLM_API_KEY=opencode-local
LLM_BASE_URL=http://127.0.0.1:4097/v1
LLM_MODEL_NAME=gpt-5.2

OPENCODE_SERVER_URL=http://127.0.0.1:4096
OPENCODE_SERVER_USERNAME=opencode
OPENCODE_SERVER_PASSWORD=your_local_password_here
OPENCODE_MODEL_PROVIDER_ID=openai
OPENCODE_MODEL_ID=gpt-5.2
OPENCODE_SOURCE_DIR=../opencode
OPENCODE_PROXY_PORT=4097
```

2. 그런 다음 바로 실행합니다:

```bash
npm run dev:opencode
```

이는 다음을 동시에 시작합니다:
- 로컬 `opencode serve`
- OpenAI-compatible 프록시（`http://127.0.0.1:4097/v1`）
- MiroFish backend
- MiroFish frontend

> 설명：`gpt-5.2`는 OpenCode의 session API를 통해 중계되므로, `response_format=json_object`는 프록시 계층에서 가능한 한 출력이 제약되고 정리됩니다.

#### 원격 OpenCode Serve 사용

로컬 `opencode serve` 대신 원격에 이미 떠 있는 OpenCode에 연결하려면 아래 모드를 사용합니다:

```bash
npm run dev:opencode-remote
```

이 모드는 다음을 동시에 시작합니다:
- 원격 OpenCode로 전달하는 로컬 OpenAI-compatible 프록시（기본 `http://127.0.0.1:4098/v1`）
- MiroFish backend
- MiroFish frontend

기본 원격 서버는 `https://ryann-oceanlike-toxophily.ngrok-free.dev/`이며, Basic Auth 값은 추적되지 않는 `.env`의 `OPENCODE_REMOTE_SERVER_USERNAME`, `OPENCODE_REMOTE_SERVER_PASSWORD`에 둡니다. 모델/포트도 필요하면 `OPENCODE_REMOTE_*` 환경 변수로 덮어쓸 수 있습니다.

#### 2. 의존성 설치

```bash
# 모든 의존성 원클릭 설치(루트 디렉터리 + 프론트엔드 + 백엔드)
npm run setup:all
```

또는 단계별로 설치:

```bash
# Node 의존성 설치(루트 디렉터리 + 프론트엔드)
npm run setup

# Python 의존성 설치(백엔드, 가상환경 자동 생성)
npm run setup:backend
```

#### 3. 서비스 시작

```bash
# 프론트/백엔드를 동시에 시작(프로젝트 루트에서 실행)
npm run dev
```

**서비스 주소：**
- 프론트엔드：`http://localhost:3001`
- 백엔드 API：`http://localhost:5001`

**개별 시작：**

```bash
npm run backend   # 백엔드만 시작
npm run frontend  # 프론트엔드만 시작
```

### 2) Docker 배포

```bash
# 1. 환경 변수 설정(소스 배포와 동일)
cp .env.example .env

# 2. 이미지 가져오기 및 시작
docker compose up -d
```

기본적으로 루트 디렉터리의 `.env`를 읽고, 포트 `3001（프론트엔드）/5001（백엔드）`를 매핑합니다

> `docker-compose.yml`에 주석으로 가속 미러 주소를 제공해두었으며, 필요에 따라 교체할 수 있습니다
