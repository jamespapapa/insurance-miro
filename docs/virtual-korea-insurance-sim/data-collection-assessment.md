# Data Collection Assessment for Virtual Korea Insurance Simulation

이 문서는 보험상품 실험용 가상 대한민국 시뮬레이터에 필요한 데이터 수집 대상을 먼저 정리하고, 각 대상의 수집 가능 여부와 그 사유를 정리한 실행 문서다.

기준 시점: 2026-03-22

## 상태 기준

- `즉시 수집 가능`: 공개 웹/파일/API로 바로 확보 가능
- `부분 수집 가능`: 공개 원천은 있으나 자동화나 구조화가 추가로 필요
- `승인/신청 필요`: 공개 설명은 있지만 실제 데이터 이용에 심사, 계약, 키 발급, 비용, 원격분석 환경 등이 필요
- `비공개/사내 필요`: 공개자료만으로는 실질 수집이 어렵고 사내 또는 유료 라이선스가 필요

## 우선 수집 결론

공개 데이터만으로도 아래는 바로 시작할 수 있다.

1. 인구·가구 규모와 지역 분포
2. 건강행태와 지역 건강지표
3. 생보 업권의 상품군/채널/지역 실적
4. 디지털 채널 친숙도 관련 공개 보고서
5. 거시·가계부채·규제 문서

정확한 판매 예측으로 가려면 결국 아래가 추가로 필요하다.

1. 실제 견적-상담-청약-가입 퍼널
2. 유지·실효·해지 이력
3. 청구·손해율
4. 경쟁상품 구조화 DB
5. 채널별 CAC/LTV와 브랜드 인식 조사

## 지금 바로 쓸 수 있는 1차 조합

### 공개 데이터만으로 가능한 MVP 입력

- MOIS 주민등록 인구·세대현황
- KDCA 지역건강통계
- KLIA 연간/월간 업권 통계
- KISA 인터넷 이용행태 보고서
- BOK 가계신용/거시 보도자료
- law.go.kr / FSC / FSS 규제 문서

### 공개 데이터만으로는 부족한 영역

- 실제 보험 보유계약 구조의 세밀한 개인 단위 분포
- 심사 승인/거절 로직의 실제 분포
- 유지·실효·해지율의 상품/채널/세그먼트 조합
- 손해율/청구빈도의 실제 상품 단위 교정
- 채널별 CAC/LTV 및 비교플랫폼 퍼널

## 다음 우선순위

1. KOSIS 표 ID를 고정해서 가계금융복지조사, 생명표, 소비통계를 자동 수집 가능 상태로 만든다.
2. SGIS OpenAPI 인증키를 발급받아 공간 레이어를 붙인다.
3. NHIS/HIRA 신청 여부를 결정하고, 승인 기반 데이터는 별도 파이프라인로 분리한다.
4. 보험사 상품요약서/약관 수집기를 만들어 경쟁상품 feature DB를 만든다.
5. 이후 사내 퍼널/유지/손해 데이터를 결합해 calibration 단계로 넘어간다.
