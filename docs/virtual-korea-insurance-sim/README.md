# Virtual Korea Insurance Simulation — Research Notes

이 폴더는 **“가상의 대한민국을 시뮬레이션하고, 그 안에서 보험상품을 실험/예측한다”**는 목표를 위한 사전 조사 문서 모음이다.

## 한 줄 결론

가능하다. 다만 이 프로젝트는 단순 SNS 여론 시뮬레이션이 아니라, 아래 6개 레이어를 합친 **시장/가계/건강/규제 디지털 트윈**에 가깝다.

1. 인구·가구 레이어
2. 소득·자산·지출 레이어
3. 건강·의료이용·위험 레이어
4. 보험상품·보험사·경쟁 레이어
5. 채널·디지털 행동 레이어
6. 거시경제·규제·이벤트 레이어

## 핵심 판단

### 1) 공개데이터만으로도 MVP는 가능
아래 정도는 공공/공개 자료로 시작할 수 있다.

- 인구·가구 구조
- 지역 분포
- 소득/자산/부채 분포
- 건강/의료이용 분포
- 생명/손해보험 산업 통계
- 채널별 일부 통계
- 거시경제 지표

이 정도면 **synthetic Korea (가상의 한국 가구군)** 를 만들고,
그 안에서 **상품 반응·가입·유지·해지**까지 보는 1차 모델은 가능하다.

### 2) 하지만 “정확한 사업 예측”으로 가려면 사내/라이선스 데이터가 필요
정확도를 올리려면 결국 아래가 필요하다.

- 실제 견적→청약→심사→가입 전환 데이터
- 유지율/실효율/해지율
- 청구 빈도와 손해율
- 채널별 CAC/LTV
- 경쟁상품 세부 구조와 가격 룰

즉 **공개데이터는 세계를 만들고**, **사내 데이터는 모델을 교정(calibration)하는 역할**을 한다.

## 추천 개발 순서

### Phase 0 — 공개데이터 기반 시장 모형
목표: “가상의 대한민국 보험시장” 초안 만들기

- synthetic household / individual 생성
- 보험사/경쟁사/채널/규제기관 agent 생성
- 상품 노출 → 고려 → 비교 → 가입 → 유지/해지 흐름 구현
- 월 단위 KPI 출력

### Phase 1 — 건강/위험도 정교화
목표: 건강상태와 의료이용을 더 실제처럼 반영

- 연령·성별·지역별 질병/의료이용 반영
- 보험 필요도/구매의향/청구확률 개선

### Phase 2 — 회사별 보정
목표: 실제 사업 의사결정에 가까운 예측력 확보

- 사내 CRM/청약/유지/손해 데이터 반영
- 경쟁사 가격/상품 변화 대응 로직 강화

## 중요한 설계 원칙

### 사람을 “개별 실명 국민”으로 만들 필요는 없음
처음부터 5천만명 1인 1에이전트로 갈 필요는 없다. 오히려 다음이 더 현실적이다.

- 1,000~50,000개의 synthetic household/individual 샘플
- 지역/연령/소득/직업/가족구성/건강위험별 대표 군집
- 보험사/GA/비교플랫폼/규제기관 같은 조직 에이전트

### 시뮬레이션 단위는 “월”이 적합
보험상품 실험은 SNS처럼 분/시간 단위가 아니라 **주/월/분기** 단위가 더 적합하다.

- 광고 노출
- 비교/상담
- 가입
- 유지/실효/해지
- 청구
- 경쟁사 대응

이런 것들이 월 단위로 누적되는 구조가 맞다.

## 핵심 질문 예시

이 시뮬레이터는 예를 들면 이런 질문에 답하도록 설계하면 된다.

- 30대 맞벌이/유자녀 가구에서 이 상품 전환율이 얼마나 나올까?
- 온라인 직판 vs 설계사 채널 중 어느 쪽이 더 효율적일까?
- 경쟁사가 가격을 낮추면 점유율이 얼마나 흔들릴까?
- 경기 둔화/의료비 상승/규제 변경이 유지율에 어떤 영향을 줄까?
- 12개월 기준 예상 가입자수, 유지율, 손해율, 수익성은 어떨까?
