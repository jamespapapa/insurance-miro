# Data Requirements for a Virtual Korea Insurance Simulation

이 문서는 **가상의 대한민국 시뮬레이터 안에서 보험상품을 실험**하려면 어떤 데이터가 필요한지, 그리고 각 데이터가 모델에서 무슨 역할을 하는지 정리한다.

---

## 0. 먼저 정리: 무엇을 예측하려는가?

데이터 요구사항은 결국 목표 KPI에 의해 정해진다.

이 프로젝트의 대표 KPI는 다음과 같다.

- 인지도 / 고려율
- 견적 요청률
- 상담 전환율
- 청약률 / 가입률
- 심사 승인률
- 유지율 / 실효율 / 해지율
- 청구 빈도 / 손해율
- CAC / LTV / 수익성
- 채널별 효율
- 세그먼트별 점유율 변화

즉 필요한 데이터는 단순한 “한국 설명 자료”가 아니라, 위 KPI를 설명할 수 있는 **행동·위험·경제 여건 데이터**여야 한다.

---

## 1. 인구·가구 기초 데이터

### 왜 필요한가
가상의 대한민국을 만들려면 가장 먼저 **사람/가구의 기본 분포**가 필요하다.

### 필요한 항목

- 연령
- 성별
- 지역(시도/시군구)
- 가구원 수
- 혼인/출산/가족 단계
- 주거형태
- 내국인/외국인 여부
- 직업/산업 대분류
- 교육 수준

### 모델에서의 역할

- synthetic household / individual 생성
- 지역별 시장 크기 추정
- 연령/가구단계별 보험 니즈 분화
- 채널/디지털 선호의 기반 변수

### 우선순위
**최우선 (필수)**

### 주요 후보 소스

- 인구주택총조사
- KOSIS 국가통계포털
- SGIS Plus

---

## 2. 소득·자산·부채·소비 데이터

### 왜 필요한가
보험은 “니즈”만으로 안 사고, **감당 가능한가**가 중요하다.

### 필요한 항목

- 가구소득 / 처분가능소득
- 금융자산 / 실물자산
- 부채 규모
- 월 고정지출
- 소비 항목 구조
- 고용 상태 / 소득 변동성
- 자영업 여부

### 모델에서의 역할

- 보험료 부담 가능성
- 가격 민감도 추정
- 유지/실효/해지 확률 추정
- 업셀/크로스셀 가능성 추정

### 우선순위
**최우선 (필수)**

### 주요 후보 소스

- 가계금융복지조사 (KOSIS)
- 한국노동패널조사(KLIPS)
- 기타 가계동향/가계지출 관련 통계

---

## 3. 건강·질병·의료이용·사망 리스크 데이터

### 왜 필요한가
보험상품, 특히 건강/질병/상해/실손/정기/종신 계열은 **건강 리스크**를 반영하지 않으면 시뮬레이션 가치가 급격히 떨어진다.

### 필요한 항목

- 연령·성별·지역별 질병 유병률
- 만성질환 보유율
- 입원/외래/약제 이용 빈도
- 건강검진·생활습관 지표
- 사망률 / 기대수명
- 의료비 지출 분포
- 질병 발생 후 이용 행태

### 모델에서의 역할

- 보험 필요도 점수 계산
- 상품별 예상 가입 타깃 정의
- 청구 확률 / 손해율 추정
- 건강 상태에 따른 심사 승인/거절 시뮬레이션

### 우선순위
**최우선 (필수)**

### 주요 후보 소스

- NHIS 데이터
- HIRA 청구/의료이용 데이터
- KDCA 지역사회건강조사 / 지역건강통계
- K-CHSI (Korean Community Health Status Indicators)

### 주의사항

- NHIS/HIRA는 매우 강력하지만 접근 절차와 제약이 있다.
- 공개자료만으로는 개인 단위 정밀 예측이 어렵고, 보통 집단 분포 기반 synthetic assignment가 필요하다.

---

## 4. 기존 보험 가입/보장 상태 데이터

### 왜 필요한가
신상품이 팔리려면 현재 보장 공백과 중복보장을 알아야 한다.

### 필요한 항목

- 보험 보유 여부
- 보험 종류별 가입률
- 연령/소득/지역별 보험 가입 패턴
- 평균 보험료 부담
- 중복보장 / 공백 영역
- 해지/실효 경험

### 모델에서의 역할

- “이미 충분히 가입한 집단”과 “보장 공백 집단” 분리
- 타깃 세그먼트 정의
- 신규 가입 vs 대체 가입 분해
- 업셀/전환 시뮬레이션

### 우선순위
**상 (필수에 가까움)**

### 주요 후보 소스

- 생명보험협회(KLIA) 통계
- 손해보험협회/관련 업권 통계
- 금융감독원 FISIS
- 보험개발원(KIDI)
- 일부 소비자 조사/패널 데이터

### 현실적 한계
정교한 보유계약 구조는 공개데이터만으로는 부족할 가능성이 크다. 초기에선 통계 기반 분포를 만들고, 이후 사내 데이터로 교정하는 방식이 현실적이다.

---

## 5. 보험사·상품·경쟁사 데이터

### 왜 필요한가
가상의 한국 시장에서 상품을 실험하려면 **우리 상품뿐 아니라 경쟁 환경**도 모델링해야 한다.

### 필요한 항목

- 경쟁 보험사 목록
- 상품 카테고리별 포지셔닝
- 보험료 수준 / 보장 구조
- 심사 정책 / 가입 가능 연령 / 특약 구조
- 판매 채널 구성
- 상품 개정 주기
- 브랜드 인지도 / 신뢰도

### 모델에서의 역할

- 비교견적 / 대체효과 / 점유율 이동 시뮬레이션
- 경쟁사 대응 로직
- 가격 인하/보장 강화 반응 시나리오

### 우선순위
**최우선 (필수)**

### 주요 후보 소스

- FISIS / 공시자료
- KLIA / 업권 통계
- KIDI 산업 통계
- 비교사이트 / 상품 공시 / 약관
- 사내 경쟁상품 DB (가능 시)

### 현실적 한계
상품 비교는 텍스트 정규화와 구조화가 필요하다. 약관/요약서/공시문서에서 상품 feature extraction 파이프라인이 있으면 매우 유리하다.

---

## 6. 채널·영업·마케팅 데이터

### 왜 필요한가
보험은 상품 자체보다도 **어떤 채널로 팔리느냐**가 결과를 크게 바꾼다.

### 필요한 항목

- 설계사 / GA / 온라인 / 제휴 / TM 비중
- 채널별 전환율
- 채널별 평균 CAC
- 채널별 유지율 차이
- 지역별/연령별 채널 선호
- 디지털 사용성 / 온라인 금융 친숙도
- 캠페인 노출과 리드 생성량

### 모델에서의 역할

- 채널 믹스 최적화
- 직판 vs 설계사 실험
- 디지털 전환 시나리오
- 비용 대비 성과 비교

### 우선순위
**상**

### 주요 후보 소스

- KLIA 통계 중 distribution channel 관련 자료
- KISA 인터넷 이용실태조사
- 전자상거래/모바일 이용 통계
- 사내 마케팅/영업 데이터

### 현실적 한계
공개자료만으로는 보험 영업 퍼널이 충분히 안 나올 수 있다. MVP에서는 업권 통계 + 합리적 가정으로 시작하고, 실제 영업 데이터를 연결해 교정하는 것이 좋다.

## 7. 규제·제도 데이터

### 왜 필요한가
보험은 규제 영향을 크게 받는다.

### 필요한 항목

- 상품 규제 / 판매 규제
- 광고/설명의무 관련 규제
- 세제 혜택 여부
- IFRS17/K-ICS 환경 변화
- 금융소비자보호 관련 제도
- 실손/건강보험 정책 변화

### 모델에서의 역할

- 특정 상품 출시 가능 여부
- 수익성 제약 조건
- 설계사 행동 변화
- 규제 이벤트 시나리오 생성

### 우선순위
**중~상**

### 주요 후보 소스

- 금융위원회(FSC)
- 금융감독원(FSS)
- 보건복지부 / 질병관리청(상품 관련 정책 영향)
- 보험개발원 / 보험연구원 보고서

## 8. 거시경제·지역경제 데이터

### 왜 필요한가
보험시장은 경기와 금리, 고용, 부동산, 지역경제 여건에 따라 민감하게 변한다.

### 필요한 항목

- 금리
- 물가 / 의료비 인플레이션
- 실업률 / 고용상태
- 자영업 경기
- 지역별 경제력 격차
- 인구 고령화 추세

### 모델에서의 역할

- 구매 여력 변화
- 유지율/해지율 변화
- 장기 보험 니즈 변화
- 경기침체/회복 시나리오

### 우선순위
**상**

### 주요 후보 소스

- KOSIS
- 한국은행
- FSS/FSC 발표자료
- 지역경제 통계

## 9. 사회심리·신뢰·브랜드·금융이해도 데이터

### 왜 필요한가
보험 구매는 순수 가격 문제만이 아니라 **신뢰, 불안, 이해도, 선호 채널**에도 좌우된다.

### 필요한 항목

- 보험/금융상품 신뢰도
- 브랜드 인지도
- 디지털 금융 친숙도
- 가족 보호 성향 / 위험회피 성향
- 상담 선호 여부
- 커뮤니티/입소문 영향도

### 모델에서의 역할

- 브랜드별 선택 확률 보정
- 온라인 구매 가능성 보정
- 마케팅 메시지 반응도 보정

### 우선순위
**중**

### 주요 후보 소스

- 공개 소비자 조사
- 디지털 이용 실태 조사
- 사내 설문/브랜드 트래커
- 외부 시장조사

### 주의사항
이 부분은 공개데이터보다 사내 설문이나 외부 리서치 구매가 더 현실적일 수 있다.

## 10. 실제 사업 결과 데이터 (가장 중요하지만 보통 비공개)

### 왜 필요한가
이 데이터가 있어야 시뮬레이터가 단순 세계관 엔진이 아니라 **사업 의사결정 엔진**이 된다.

### 필요한 항목

- 리드 → 상담 → 청약 → 계약 전환율
- 상품별 유지율 / 실효율 / 해지율
- 심사 승인/보완/거절율
- 사고/질병/청구 빈도
- 손해율 / 사업비율 / 수익성
- 채널별 CAC/LTV
- 코호트 성과

### 모델에서의 역할

- calibration (보정)
- backtesting (과거 검증)
- scenario confidence 개선

### 우선순위
**장기적으로 절대 필수**

### 소스

- 사내 CRM
- 사내 청약/계약/유지/해지/청구 데이터
- 제휴 채널 로그

### MVP에 반드시 필요한 것

7. 경쟁상품 세부 구조
8. 사내 전환/유지/손해 데이터

### 후반 고도화용

9. 이벤트 반응 데이터
10. 커뮤니티/입소문 데이터
11. 세밀한 의료코드 기반 질환 세분화

## 12. 추천 구축 방식

### 공개데이터 기반 1차 구축
- Census/KOSIS/SGIS로 synthetic population 생성
- NHIS/HIRA/KDCA로 health risk 분포 부여
- KLIA/KIDI/FSS로 보험시장 구조 부여
- 월 단위 가입/유지/해지/청구 시뮬레이션 구현

### 2차 교정
- 사내 영업 퍼널/청약/유지 데이터 반영
- 경쟁상품 데이터 정규화
- 채널별 실측 파라미터 보정

### 3차 고도화
- 마케팅 실험 결과 반영
- 정책 이벤트 / 규제 변화 / 경기 시나리오 엔진 결합

## 13. 핵심 메모

이 프로젝트에서 진짜 중요한 건 데이터를 “전부 다 모은다”가 아니라:

- 무엇을 **agent state** 로 넣을지,
- 무엇을 **의사결정 변수** 로 쓸지,
- 무엇을 **결과 KPI** 로 평가할지,
- 무엇을 **공개데이터로 만들고**, 무엇을 **사내 데이터로 보정할지**

를 분리하는 것이다.

즉 필요한 데이터는 크게 아래 둘로 나뉜다.

1. **세계를 만드는 데이터**
2. **모델을 맞추는 데이터**

보험 실험용 디지털 트윈은 둘 다 필요하지만, 시작은 1번만으로도 충분히 가능하다.
